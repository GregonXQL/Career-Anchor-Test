# M5 企业主体提审材料

## 建议服务类目

主类目建议选择：**工具 → 信息查询**（以微信公众平台提交时显示的最新类目为准）。

选择理由：小程序的实际功能是用户填写职业价值观问卷，服务端按固定规则计分，用户查询自己的历史结果和职业锚解释。它不提供在线课程、职业技能培训、医疗服务、心理咨询或诊断，因此不应选择“教育 → 职业培训”或医疗/心理相关类目。

如果平台审核反馈要求调整类目，应以审核反馈及企业营业执照经营范围为准，不通过修改页面文案伪装实际功能。

## 小程序功能介绍

可直接粘贴：

> 本小程序提供职业锚价值观问卷工具。用户经微信登录并输入邀请码后完成 40 道题，系统依据固定规则生成八维结果、职业价值排序及文字解释，并支持查询个人历史报告。结果仅用于个人职业探索参考，不构成医疗、心理诊断、职业资格评价或决策建议。

## 审核人员操作路径

可直接粘贴，并在提交前替换方括号内容：

> 打开小程序后阅读隐私提示并点击“同意并继续” → 点击“微信一键进入” → 首页点击“开始测评” → 输入审核邀请码 `[提交前生成]` → 完成 40 道题及 3 道附加题 → 提交后查看职业锚报告。首页和报告底部均展示免责声明。历史报告入口位于首页第二张卡片。

审核邀请码要求：

- 提交前由管理控制台“生成邀请小程序码”功能创建一个 `maxUses` 至少为 10、有效期覆盖完整审核周期的邀请；
- 将生成结果中的 8 位 code 填入上述审核说明，审核人员可直接手动输入，不要求扫码；
- 审核完成后在管理控制台停用该邀请；
- 不提供管理员密码，审核普通用户完整路径即可。

## 隐私保护指引声明项

按实际数据处理情况声明：

- 微信用户身份标识：用于登录、区分用户及查询本人历史报告；
- 用户填写的信息：40 道题答案和 3 道附加题选择，用于服务端计分；
- 测评结果：八维分数、前三项职业锚及创建时间，用于展示本次和历史报告；
- 昵称由用户通过微信昵称输入组件主动选择或填写，头像由用户通过微信头像组件、相册或相机主动选择，用于区分并展示测评报告。

隐私指引中不应声明未实际使用的通讯录、手机号、精确位置等信息。应声明保存图片到相册（报告长图、邀请小程序码）和写入剪贴板（复制邀请码），并写明仅在用户或管理员主动操作时调用。

## 提审前页面检查

- 启动页：隐私提示可查看、同意和拒绝；拒绝后不发起登录。
- 测评说明页：明确量表流程和“仅供参考”免责声明。
- 报告页：使用“职业价值雷达”，不使用疾病、障碍、治疗、诊断等表述。
- 首页和报告页：固定展示“不构成任何诊断或决策依据”。
- 空数据和网络失败：有空态、错误提示和重试入口，不出现白屏。
- 正式环境：接口使用已备案 HTTPS 域名，开发地址和调试开关不进入上传版本。
